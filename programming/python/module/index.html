    模块

    01. 导入模块的基本语法

    使用import 可以导入模块，基本使用语法有以下两种：

    • import 模块名[as 别名],模块2[as 别名]...：使用这种语法，会导入模块中的所有成员，包括(变量、函数、类等）。以这种语法导入的模块，需要使用模块中的成员时，需以模块名（或别名）为前缀。
    • from 模块名 import 成员名1 [as 别名1]，成员名2 [as 别名2]，…： 使用这种语法格式的 import 语句，只会导入模块中指定的成员，而不是全部成员。同时，当程序中使用该成员时，无需附加任何前缀，直接使用成员名（或别名）即可

    语法中[] 的内容，可以省略。

    按照PEP8的语法规范，导入多个模块，不建议使用逗号隔开的方式，应该导入每个模块都独占一行

    02.import导入整个模块

    import 语句会导入整个模块，可以通过as 指定别名。使用成员时需以模块名或别名作为前缀。使用方式如下：

    import sys

print(sys.argv[0])
>>>

    import sys as s

print(s.argv[0])
>>>

    • 一次性导入多个模块，模块之间以, 分割：
    import sys as s,os as o

print(s.argv[0])
print(o.sep)
>>>

    03. from...import

    使用from 模块名 import 成员的方式，可以导入模块中的指定成员，使用时，无需以模块名作为前缀：

    from sys import argv

print(argv[0])
>>>

    from sys import argv as a

print(a[0])

    from sys import argv as a, winver as mv

    04. import *

    import * 或 from 模块名 import * 的方式可以导入模块下的所有成员，但不推荐使用，因为如果多个模块中有同名的成员，会报错

    05. 导入自定义模块

    每一个以.py 为后缀名的文件都是一个模块，都可以被导入到其他文件中使用:

    • 创建一个名为demo.py的文件，并编写以下代码：
    # 创建一个模块文件

# 全局变量
name = "小明"
age = 19

# 函数
def say_hello():
    print(name, age)

# 类
class XiaoMing(object):
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def say(self):
        print("我的名字叫{},我的年龄是{}岁".format(self.name, self.age))
    • 再新建一个test.py 的文件，并导入demo.py：
    import demo

demo.say_hello()
a = demo.XiaoMing("小花",20)
a.say()
>>>
小明 19
我的名字叫小花,我的年龄是20岁

    06. 模块的搜索顺序

    python解释器，在导入模块时，搜索顺序如下：

    1. 优先搜索当前目录下的同名模块，如果有就直接导入；
    2. 如果没有，再搜索系统目录

    在给文件取名时，不能和系统的模块文件重名，否则会出现不必要的错误 

    import  random as rd

rand = rd.randint(0,10)
print(rand)
>>>
2

    以上代码演示了，导入系统模块random ，并使用random的内置函数randint 在0-10之间生成随机数。

    • random 是系统模块，所以程序执行正常;
    • 如果在这个文件内有一个文件，文件名是random.py。运行以上代码则会报错，因为python解释器导入模块会优先搜索当前目录同名模块

    6.1 __file__ 查看模块的完整路径

    __file__ 内置属性，可以查看导入模块的完整路径，对于上面出现的问题，可以使用这个属性验证

    import  random

print(random.__file__)
>>>
/Users/kaka/Documents/python/study/random.py
    • 如果删除掉当前目录下的random.py文件，再运行以上代码：
    import  random

print(random.__file__)
>>>
/Library/Frameworks/Python.framework/Versions/3.6/lib/python3.6/random.py

    07.__doc__ 模块文档

    和函数或类一样，也支持为模块自定义文档。

    • 在模块开头位置，以文档注释的方式添加字符串说明即可：
    """
 name:字符串格式
 age:int格式
 say_hello:函数
 XiaoMing：类
"""

# 全局变量
name = "小明"
age = 19

# 函数
def say_hello():
    print(name, age)

# 类
class XiaoMing(object):
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def say(self):
        print("我的名字叫{},我的年龄是{}岁".format(self.name, self.age))

    7.1 查看文档

    在导入模块后，可以使用内置属性__doc__ 查看模块的说明文档

    import demo

print(demo.__doc__)
>>>
name:字符串格式
age:int格式
say_hello:函数
XiaoMing：类

    08. __name__

    导入模块时解释器会把模块文件执行一次，如果模块文件存在测试代码，在导入模块后，直接执行，也会出现原模块的测试代码：

    """
 name:字符串格式
 age:int格式
 say_hello:函数
 XiaoMing：类
"""

# 全局变量
name = "小明"
age = 19

# 函数
def say_hello():
    print(name, age)

# 类
class XiaoMing(object):
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def say(self):
        print("我的名字叫{},我的年龄是{}岁".format(self.name, self.age))

say_hello()
a = XiaoMing("小花",20)
a.say()
>>>
小明 19
我的名字叫小花,我的年龄是20岁
    • 在新的文件内导入以上模块后运行文件：
    import demo

>>>
小明 19
我的名字叫小花,我的年龄是20岁

    由于导入模块后，并不需要执行模块中的测试代码，所以可以使用python的内置属性__name__ 解决：

    • 当__name__ 在模块文件中，默认返回是字符串"__main__"
    • 当模块导入到新文件后，__name__ 属性会变为模块的文件名
    def say_hello():
    print("这是模块文件")

say_hello()
print(__name__)
>>>
这是模块文件
__main__
    • 模块导入到test.py后：
    import demo

print("这是导入后的文件")
>>>
demo
这是导入后的文件

    8.1 if __name__ == __main__

    为了解决模块的测试代码不在导入后的文件出现的问题，可以在模块文件内加入if __name__ == __main__ 的判断，并将测试代码，放在判断内部

    def say_hello():
    print("这是测试代码的显示")

if __name__ == "__main__":
    print(__name__)
    say_hello()
>>>
__main__
这是测试代码的显示
    • 这样之后，再讲以上模块导入到新文件后，就不会出现测试代码
    import demo

print("这是导入后的文件")
>>>
这是导入后的文件

    8.2 自定义main函数

    制作模块是，可以自定义main函数，将所有的测试代码都放在函数内部，并加以下代码：

    def say_hello():
    print("这是测试代码的显示")

def main():
    print(__name__)
    print("这是模块文件的测试函数内部")
    say_hello()

if __name__ == "__main__":
    main()

    09 .包的概念

    • 包是一个包含多个模块的特殊目录
    • 目录下有一个 特殊的文件 __init__.py
    • 包的命名方式和变量名一样，使用小写字母+_ 的方式，可以使用数字，但不能以数字开头，比如ka_01,ka_package

    10. pycharm 创建包的两种方式

    1. 新建文件夹，在文件夹内创建__init__.py 文件，文件内可以不写任何代码
    2. 新建python_Package ,pycharm会自动创建包和__init__.py 文件

    11. 包的导入

    包的本质还是模块，所以导入包的语法和导入模块的语法类似：

    1. import 包名 [.模块名][as 别名]
2. from 包名 import 模块名 [as 别名]
3. from 包名.模块名 import 成员名 [as 别名]
    • 定义一个包，如下
    | ka_message
|--__init__.py
|--send_message.py
|--receive.py

    11.1 import 包名 [.模块名][as 别名]

    # 1.导入整个包
import ka_message

# 2. 导入包内某个模块
import ka_message.send_message

# 3.设置别名
import ka_message.send_message as module

    11.2 from 包名 import 模块名 [as 别名]

    # 1.导入某个模块
from ka_message import send_message

# 2.设置别名
from ka_message import send_message as module

    11.3 from 包名.模块名 import 成员名 [as 别名]

    # 导入包内某个模块的某个成员
from ka_message.send_message import send

# 导入包内某个模块的某个成员,并设置别名
from ka_message.send_message import send as send_foo

    12. 制作模块

    如果希望将自己制作的模块分享给他人，可以使用模块制作的方式，共有三步

    1. 在包内创建setup.py文件

    from distutils.core import setup

setup(name="ka_message",  # 包名
      version="1.0",  # 版本
      description="kaka's 发送和接收消息模块",  # 描述信息
      long_description="完整的发送和接收消息模块",  # 完整描述信息
      author="kaka",  # 作者
      author_email="hmz270@163.com",  # 作者邮箱
      url="无",  # 主页
      py_modules=["ka_message.send_message",
                  "ka_message.receive"])

    2. 终端中构造文件

    $ python3 setup.py build

    3. 终端中生成发布压缩包

    $ python3 setup.py sdist

    12.1 安装模块

    $ tar -zxvf ka_message-1.0 tar.gz
$ sudo python3 setup.py install

    12.2 卸载模块

    $ cd 到安装目录
$ sudo rm -r ka_message*

    13. pip安装第三方模块

    pip 是python的包管理工具，用于安装第三方模块

    • 安装python3的命令是pip3 。基础语法如下：
    # 安装第三方模块
$ pip3 install 模块名

# 卸载三方模块
$ pip3 uninstall 模块名

# 查看第三方模块列表
$ pip list

# 查看标准库和第三方模块,在python命令行内：
$ help("modules")
