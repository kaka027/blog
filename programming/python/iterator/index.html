    迭代器/生成器

    01. 迭代器

    1.1 什么是迭代器

    • 当对一个可迭代对象使用iter 函数后，会返回一个可迭代对象。
    • 对于可迭代对象，可使用next 函数，去获取元素，没执行一次，获取一个。直到全部获取完毕，会抛出StopIteration 提示。表示无元素可获取
    a = [1, 2, 3]
gen = iter(a)
print(gen)
print(next(gen))
print(next(gen))
print(next(gen))
print(next(gen))
>>>
<list_iterator object at 0x7fd4a68597f0>
1
2
3
StopIteration

    1.2 迭代器遍历

    • 迭代器可以使用for循环取值和操作
    a = [1, 2, 3]
gen = iter(a)
print(gen)
for i in gen:
    print(i,end=" ")
>>>
<list_iterator object at 0x7fa898f59748>
1 2 3

    1.3 迭代器协议

    • 对比可迭代对象，迭代器 的内部只是多了一个__next__() 函数。正因为有它，才可以使用next 来获取元素
    class Array:
    index = 0
    mylist = [1, 2, 3]

    # 返回该对象的迭代器类的实例
    # 因为自己就是迭代器，所以返回self
    def __iter__(self):
        return self
    # 当无元素时，必须抛出StopIteration
    def __next__(self):
        if self.index <= len(self.mylist)-1:
            value = self.mylist[self.index]
            self.index +=1
            return value
        raise StopIteration

my_iterator = iter(Array())
print(next(my_iterator))  # 1
print(next(my_iterator))  # 2
print(next(my_iterator))  # 3
print(next(my_iterator))  # StopIteration

    0.2 生成器

    2.1 生成器的定义和语法

    • 生成器（英文名 Generator ），生成器的本质也是迭代器的一种。可以使用for 循环和next 方法来获取元素。
    • 生成器的出现，实现了延时计算，可以避免大量数据直接加载到内存的消耗。

    2.2 生成器的创建

    • 列表推导式
    m = [x for x in range(4)]
print(m)
>>>
[0, 1, 2, 3]
    • 当列表生成式的[] 改为() 。返回的就不是列表，而是生成器了
    m = (x for x in range(4))
print(m)
>>>
<generator object <genexpr> at 0x7fb42978ae60>

    2.3 使用yield

    yield 相当于函数内的return ，但与return 有以下区别：

    • 当函数运行到yield 时，函数运行会暂停挂起，并把yield 接收的值返回。
    • 若yield 没有接收任何值，则返回None
    • yield 虽然返回了，但是函数运行并没有结束，当使用next 方法时，会先执行yield 之后的代码。
    def generator(n):
    index = 0
    while index <5:
                index += 1
        yield index
        print("这是yield以后的代码，此时yield值为：",index)

m = generator(6)
print(m)
>>>
<generator object generator at 0x7fabc1292e60>

    以上实例中，m=genertor(6) 执行时，函数并没有激活，且此时m 是一个生成器对象。当使用next 或者for 循环时，生成器函数才开始激活运行。

    2.4 生成器的使用

    生成器的取值和列表切片的方式不同，要获取生成器中的元素，只有两种方式：

    1. 使用next 方法，依次获取生成器内的元素，当生成器内没有元素时，会抛出StopIteration 错误
    def generator(n):
    index = 0
    while index < 3:
        index +=1
        yield index
        print("这是yield以后的代码，此时yield值为：", index)

m = generator(3)
print(next(m))
print(next(m))
print(next(m))
print(next(m))
>>>
1
这是yield以后的代码，此时yield值为： 1
2
这是yield以后的代码，此时yield值为： 2
3
这是yield以后的代码，此时yield值为： 3
StopIteration
    1. 使用for 循环遍历生成器
    def generator(n):
    index = 0
    while index < 3:
        index += 1
        yield index
        print("这是yield以后的代码，此时yield值为：", index)

m = generator(3)
for i in m:
    print(i)
>>>
1
这是yield以后的代码，此时yield值为： 1
2
这是yield以后的代码，此时yield值为： 2
3
这是yield以后的代码，此时yield值为： 3

    2.5 生成器的状态

    生成器在其生命周期中，会有如下四个状态：

    • GEN_CREATED ：生成器已被创建，还未激活
    • GEN_RUNING ： 解释器正在执行（只有在多线程应用中才能看到这个状态）
    • GEN_SUSPENDED : 在yield 表达式处暂停
    • GEN_CLOSED : 生成器执行结束
    from inspect import getgeneratorstate

def generator(n):
    index = 0
    while index < 3:
        index += 1
        yield index
        # print("这是yield以后的代码，此时yield值为：", index)

m = generator(4)
print(getgeneratorstate(m))
print(next(m))
print(getgeneratorstate(m))
print(next(m))
print(getgeneratorstate(m))
print(next(m))
print(getgeneratorstate(m))
print(next(m))
print(getgeneratorstate(m))
>>>
GEN_CREATED
1
GEN_SUSPENDED
2
GEN_SUSPENDED
3
GEN_SUSPENDED
StopIteration
GEN_CLOSED
